# feishu-doc-meeting-summary - 飞书会议纪要生成

> 自动从飞书会议内容生成结构化会议纪要。

## 功能特性

### 核心能力
1. **自动提取** - 从会议录音/文字提取要点
2. **待办提取** - 自动识别并列出待办事项
3. **责任人** - 标注每个待办的责任人
4. **时间节点** - 标注截止时间
5. **格式统一** - 按模板格式化输出

### 适用场景
- 周会/月会纪要
- 项目会议记录
- 客户沟通纪要
- 内部讨论总结

## 使用方法

### 基本使用
```
生成这次会议的纪要：
- 会议内容：xxx
- 提取待办：是
- 格式：详细版
```

### 指定模板
```
按以下格式生成：
- 参与人
- 讨论要点（3条）
- 决策事项
- 待办清单（带责任人+时间）
```

## 技术参数
- 支持会议时长：不限
- 处理速度：约 1 分钟/10 分钟录音转文字
- 提取准确率：90%+

## 使用示例

### 示例 1：周会纪要
**输入**：周会讨论内容
**输出**：
- 本周工作回顾
- 下周工作计划
- 需要协调事项

### 示例 2：项目会议
**输入**：项目讨论
**输出**：
- 决策事项
- 待办清单（任务+责任人+截止时间）

### 示例 3：客户沟通
**输入**：客户拜访记录

- 客户需求汇总

- 下次沟通时间

## 注意事项
1. 需要有会议文字内容或录音
2. 待办提取需要明确上下文
3. 建议会后 24 小时内生成
4. 可多次迭代优化
